# 知乎风格参考

## 平台特征

- 读者群体偏高知、偏理性
- 重视逻辑、数据、引用来源
- 反感营销感和"自媒体腔"
- 长内容友好，读者耐心较好
- "专业感"和"亲历感"是加分项

## 内容形式选择

根据选题特点选择发布形式：

| 形式 | 适用场景 | 标题策略 |
|------|---------|---------|
| 回答 | 选题适合以问题形式呈现时 | 找到/创建一个好问题 |
| 文章 | 原创深度内容、不适合问答形式时 | 专业感+独特视角 |

## 排版规范

### 标题

**回答体标题（问题）**：
- "如何看待{事件}？"
- "为什么{现象}？"
- "{主题}是一种什么体验？"
- "{A}和{B}有什么区别？"

**文章体标题**：
- 专业感 + 独特视角
- 避免情绪化和感叹号
- 可以用"：""——"做解释性补充

示例：
- 回答体：如何看待2026年AI Agent大规模落地？行业会发生哪些变化？
- 文章体：从技术债到组织债：创业公司规模化的隐性成本
- 亲历体：在字节跳动工作三年，我对"大厂经验"的重新理解

### 开头策略

知乎读者喜欢"先知道答案"：

```
模式1："先说结论：{一句话核心观点}。下面展开说。"
模式2："利益相关：{说明自己的身份/经验}。"
模式3：直接上干货，第一段就给出核心论点。
```

### 正文格式

```
段落长度：3-5句/段（可以比公众号长）
小标题：每800-1200字
语气：分析性、客观、可以带个人判断但要标注

引用格式（大量使用）：
> "引用内容" —— 来源
> 数据来源：{报告名/机构}，{年份}

逻辑连接词要显性：
"因此""然而""具体来说""换个角度看""本质上"

代码/公式：
- 技术内容可以用代码块
- 数学概念可以用 LaTeX 公式

表格：
- 对比数据优先用表格呈现
- 知乎 Markdown 原生支持表格

分割线：
- 大段落之间用 --- 分隔
```

### 对立观点段落

知乎读者尊重多元视角。建议在文章中加入：

```markdown
## 另一种声音

当然，也有人持不同观点。{对立观点的陈述}。

这种观点有一定道理，因为{原因}。但我认为{反驳/补充}。
```

### 文末

```markdown
---

以上。

如果这篇回答/文章对你有帮助，点个赞让更多人看到。

有不同看法也欢迎在评论区讨论，理性交流。
```

### 利益相关声明

如果文章涉及行业经验或产品推荐：
```
利益相关：我在{行业/公司}{角色}，以下观点基于个人经验，仅供参考。
```
