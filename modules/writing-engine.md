# 模块B：写作引擎 (Writing Engine)

## 概述

将选题转化为一篇高质量"母版文章"，后续由平台适配器做差异化改写。

## 写作流程

```
选题确认 → 风格匹配 → 素材收集 → 大纲生成 → 用户审核大纲
         → 全文写作 → 自检润色 → 输出母版文章
```

## Step 1: 风格匹配

四种风格体系，根据选题特征自动推荐，用户可覆盖：

| 代号 | 名称 | 适用信号 | 文章特征 |
|------|------|---------|---------|
| `deep` | 深度干货型 | 技术解析、方法论、行业分析 | 长文(3000-5000字)、小标题分段、数据表格、案例剖析、可操作建议 |
| `story` | 故事情感型 | 人物经历、社会现象、成长感悟 | 叙事驱动、场景描写、情感递进、金句点题、引发共鸣 |
| `sharp` | 观点犀利型 | 争议话题、反常识、行业批评 | 开头抛论点、正反对比、犀利表达、逻辑严密、引发讨论 |
| `light` | 轻松科普型 | 知识普及、生活技巧、工具推荐 | 口语化、比喻多、分点清晰、轻松易读、实用感强 |

自动匹配逻辑：
- 选题含争议/对立观点 → `sharp`
- 选题含人物故事/经历 → `story`
- 选题含专业知识/方法论/数据 → `deep`
- 选题偏日常/技巧/工具 → `light`
- 无法判断 → 用 AskUserQuestion 让用户选择

如果 EXTEND.md 的 `default_style` 不是 `auto`，直接使用指定风格。

## Step 2: 素材收集

使用 deep-research 方法论收集写作素材：

```
素材收集清单：
1. 核心事实：事件的来龙去脉、关键数据
2. 权威引用：专家观点、研究报告、官方数据
3. 案例故事：真实案例或典型场景（2-3个）
4. 对立观点：不同立场的声音（至少2种）
5. 金句素材：可用于点题或分享的精炼表达
6. 背景知识：读者可能不了解的前置信息
```

用 WebSearch 和 WebFetch 收集，注意：
- 优先使用近7天内的数据源
- 标注每条素材的来源和可信度
- 争议性内容需要多源交叉验证

## Step 3: 大纲生成

根据选定风格，读取对应的大纲模板：

- `deep` → [templates/outline-deep.md](../templates/outline-deep.md)
- `story` → [templates/outline-story.md](../templates/outline-story.md)
- `sharp` → [templates/outline-sharp.md](../templates/outline-sharp.md)
- `light` → [templates/outline-light.md](../templates/outline-light.md)

按模板结构生成大纲，包含：
- 每个段落的核心内容（一句话）
- 段落间的逻辑关系
- 素材分配（哪个案例/数据用在哪里）
- 金句位置标记

用 AskUserQuestion 展示大纲，选项：
- "大纲OK，开始写作"
- "调整大纲"（用户给出修改意见后重新生成）
- "换一种风格重新生成"

## Step 4: 全文写作

### 通用写作规范

```
1. 段落长度：每段3-5句，避免大段文字墙
2. 金句密度：每800-1000字埋入一句金句（方便读者截图分享）
3. 过渡自然：段落间用过渡句连接，不能硬跳
4. 案例具体：案例要有细节（人名/公司名/数字），不要泛泛而谈
5. 数据可查：引用的数据标注来源
6. 口语化程度：deep=20% | story=50% | sharp=30% | light=80%
7. 避免AI腔：不用"值得注意的是""总而言之""在当今社会"等套话
8. 标题层级：母版使用 ## 作为一级小标题，### 作为二级（预留 # 给平台标题）
```

### 爆文结构模型

无论哪种风格，都遵循"钩子→展开→收束"的三段式结构：

**钩子开头（前100-200字，决定读者是否继续）**

根据风格选择钩子类型：
- `deep`: 数据钩子 → 震撼数据或反直觉的对比开场
- `story`: 场景钩子 → 一个人的真实瞬间，有画面感
- `sharp`: 冲突钩子 → 直接抛出争议性观点或质疑
- `light`: 共鸣钩子 → "你是不是也遇到过这种情况？"

**正文展开（主体部分，按大纲推进）**

- 每个核心论点独立成"段落群"（小标题 + 2-4段正文）
- 论点之间用转折/递进/对比连接
- 案例和故事穿插在论证中间，不堆在一起
- 每个段落群结尾有一句总结性金句

**结尾收束（最后200-300字）**

根据风格选择收束方式：
- `deep`: 行动清单 → 给出3-5个可操作建议
- `story`: 情感升华 → 将个体故事拉到普遍意义
- `sharp`: 开放质问 → 抛出一个让读者思考的问题
- `light`: 总结清单 → 一句话总结 + 引导收藏

## Step 5: 自检润色

写完全文后，按以下清单自检：

```
□ 事实核查：文中引用的数据、事件、人名是否准确？
□ 逻辑链条：论点之间是否有逻辑断裂？
□ 金句检查：是否每800-1000字有一句可截图分享的金句？
□ 开头吸引力：前3句话能否让人想继续读？
□ AI腔检测：是否有"值得注意""不可否认""在当今"等AI套话？删掉或改写。
□ 字数范围：总字数是否在风格对应范围内？
  - deep: 3000-5000字
  - story: 2000-4000字
  - sharp: 2000-3500字
  - light: 1500-2500字
□ 可读性：是否有超过5句话的段落？拆分。
□ 结尾号召力：结尾是否有引导读者行动（评论/转发/收藏）？
```

发现问题自动修正，修正后输出母版文章为 `master.md`。

## 输出格式

母版文章以 Markdown 格式输出，包含 frontmatter 元数据：

```markdown
---
title: "文章标题"
topic: "选题来源/方向"
style: deep|story|sharp|light
word_count: 3500
created: 2026-04-15
tags:
  - 标签1
  - 标签2
---

## 小标题1

正文内容...

## 小标题2

正文内容...
```

保存为 `{输出目录}/{日期}_{slug}/master.md`。
