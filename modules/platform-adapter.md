# 模块C：平台适配器 (Platform Adapter)

## 概述

将母版文章改写为适合不同平台调性的版本。这是本SKILL的核心原创模块——同一个内容，在不同平台需要不同的表达方式才能获得最好的传播效果。

## 适配流程

```
读取母版文章(master.md)
→ 读取 EXTEND.md 中开启的平台列表
→ 对每个平台：
   1. 改写标题（生成3个候选）
   2. 改写内容（调整结构、语气、长度）
   3. 添加平台特有元素
   4. 输出平台版本文件
→ 用 AskUserQuestion 让用户为每个平台选择标题
```

## 平台适配规则

详细的平台风格参考见各文件：[公众号](../platform-styles/wechat.md)、[今日头条](../platform-styles/toutiao.md)、[知乎](../platform-styles/zhihu.md)、[小红书](../platform-styles/xiaohongshu.md)、[朋友圈](../platform-styles/moments.md)。以下是核心差异点。

### 微信公众号 (wechat.md)

**标题规则**：
- 长度：15-64字，越具体越好
- **标题必须以文章序号开头**：格式为 `{series_index:02d}：{标题}`，例如 `01：为什么越「菜」的人越自信？`
- 善用数字："3个方法""90%的人不知道"
- 善用符号：「」用于关键词强调
- 可用长标题+副标题形式
- 情绪词适度：不要过度标题党，但要有好奇心缺口

**wechat.md 文件结构模板（强制）**：

以下是 wechat.md 的完整结构，**必须严格遵循此顺序**，不可调换：

```markdown
![题图](images/brand-header.jpeg)

**「{系列名称}」系列 · 第{N}篇**

# {序号}：{标题}

{正文内容，含正文配图 — 正文配图是用 baoyu-article-illustrator 生成的内容插图，不是品牌图}

---

{文末CTA引导语：在看/转发/关注}

![底图](images/brand-footer.jpeg)
```

**结构要点**：
1. **品牌题图在最前面**（`![题图](images/brand-header.jpeg)`），在标题之前
2. **系列标签紧跟题图**（`**「系列名称」系列 · 第X篇**`），标明当前是系列第几篇
3. **标题带序号**（`# 01：标题内容`），序号两位数补零
4. **正文含内容配图**（由 baoyu-article-illustrator 生成，位于正文段落之间，**不是品牌图**）
5. **品牌底图在最后面**（`![底图](images/brand-footer.jpeg)`），在 CTA 引导语之后

⚠️ **特别注意**：
- 品牌题图和底图只用于文首和文末固定位置
- 正文中间的配图是独立生成的内容插图（illustrator 生成），**严禁把品牌图当作正文配图**
- 如果文章属于系列，必须有系列标签行；非系列文章可省略
- 题图和标题之间有且仅有系列标签一行（加粗居中），不要加分割线或其他内容
- 如果 EXTEND.md 中 `wechat_ads.header_image` 或 `wechat_ads.footer_image` 为空，则跳过对应的品牌图插入

**内容改写**：
- 保留母版的全部深度内容
- 段落间距加大，每段2-3句话
- 增加小标题密度（每500-800字一个小标题）
- 每个小标题可以更"文艺"或"吸睛"
- 文末添加引导语：引导在看/转发/关注
- 关键句加粗，金句可单独成段
- 外部链接转为文末引用（baoyu-post-to-wechat 会自动处理）
- **正文中需要预留配图插入位置**（用 `<!-- illustration: 描述 -->` 占位，后续由 format-engine 替换为实际图片）

**字数**：2000-5000字

**输出文件**：`wechat.md`

### 今日头条 (toutiao.md)

**标题规则**：
- 长度：15-30字，信息密度要高
- 必须包含核心信息（头条标题=内容摘要）
- 避免纯标题党（头条算法会惩罚高点击低完读的内容）
- 适合用"谁做了什么+结果/影响"的叙事结构
- 可以用设问句

**内容改写**：
- 第一段极其重要——决定算法是否推荐（必须包含核心信息+引发继续阅读的钩子）
- 段落更短更碎（每段1-3句话）
- 减少文艺化表达，增加信息密度
- 关键词布局：标题和前200字中要包含核心关键词（帮助算法识别分类）
- 适当增加小标题和分隔线（提高视觉节奏）
- 配图标注更具体（头条读者习惯看图+文）
- 文末不需要引导关注（头条逻辑不同），但可以引导评论

**字数**：1500-3000字

**输出文件**：`toutiao.md`

---

### 知乎 (zhihu.md)

**标题规则**：
- 如果适合做成"回答"，标题应该是一个好问题（"如何看待...""为什么..."" ...是一种什么体验"）
- 如果做成"文章"，标题要有专业感+独特视角
- 避免情绪化标题——知乎读者反感"营销感"
- 数字和具体信息加分："从0到10万用户，我做对了这3件事"

**内容改写**：
- 逻辑链条要更严密——知乎读者习惯质疑和追问
- 增加引用和数据来源标注（可以用 "> 引用" 格式）
- 可以更长更深，知乎读者耐心较好
- 开头用"先说结论"或"先上总结"的方式（知乎读者喜欢先知道答案）
- 语气可以更学术化、分析化
- 适合加入"对立观点分析"段落（展示深度思考）
- 文末可以用"以上"结尾，或追加"延伸阅读"
- 如果有个人经历相关，用"利益相关："开头说明

**字数**：2000-6000字

**输出文件**：`zhihu.md`

### 小红书 (xiaohongshu.md)

**标题规则**：
- 短标题：10-20字
- 必须加 emoji（1-3个，放在开头或关键位置）
- 口语化，像跟朋友说话
- 制造"发现感"和"获得感"："原来...""终于找到了...""后悔没早知道"
- 感叹号多于句号

**内容改写**：
- 从母版中提炼核心要点，大幅精简
- 分点列举（用 emoji 编号代替数字编号）
- 每点1-2句话，不展开论证
- 语气亲切口语化，像闺蜜/好友分享
- 善用 emoji 做视觉分隔
- 结尾加话题标签（#标签1 #标签2，3-5个）
- 结尾引导收藏："记得收藏！""关注我获取更多..."

**字数**：300-800字（纯文案部分）

**注意**：小红书的核心是图片而非文字。这里的文案将配合 baoyu-xhs-images 生成的图卡使用。文案主要作为帖子正文，图卡才是内容主体。

**输出文件**：`xiaohongshu.md`

### 朋友圈 (moments.md)

**定位**：公众号文章的"社交预告片"，用最短文字引导好友点开全文。

**文案规则**：
- 总字数严格控制在 80 字以内（超 99 字/6行会被折叠，阅读率骤降）
- 结构：钩子句(1句) + 核心观点(1-2句) + 行动引导(1句，用 👇 指向链接)
- 语气像朋友随手分享，不像广告。可带一句个人感受
- 禁止：营销词、@ 任何人、话题标签、裸链接

**配图建议**：
- 从文章配图中精选 1/3/6/9 张（网格整齐原则，避免 2/4/5/7/8 张）
- 单图首选封面图 cover.png
- 9 张可直接复用小红书图卡
- 可选：生成一张 1080x1080 金句卡片

**字数**：40-80字

**输出文件**：`moments.md`

## 标题选择

所有平台的标题改写完成后，用 AskUserQuestion 为每个平台选择标题：

```
问题: "请为{平台名}选择标题"
选项:
  - label: "标题A"
    description: "特点描述"
  - label: "标题B"
    description: "特点描述"
  - label: "标题C"
    description: "特点描述"
```

可以一次性为所有平台选择（multiSelect 模式），也可以逐个选择。

选择后更新各平台版本文件的标题。

## 跨平台一致性检查

适配完成后，快速检查：
1. 核心信息是否一致——不同平台版本不能出现事实矛盾
2. 案例/数据是否准确——改写过程中不能引入错误
3. 各版本是否满足字数范围要求
