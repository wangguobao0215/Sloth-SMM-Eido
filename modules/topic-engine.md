# 模块A：选题引擎 (Topic Engine)

## 概述

负责从热点新闻或模糊方向中提炼出高价值选题，并进行多维评分和推荐。

## 热点模式（场景1）

### Step 1: 抓取热点

读取 EXTEND.md 中的 `news_sources` 和 `news_keywords` 配置。

调用 news-aggregator-skill 的脚本：

```bash
# 定位 news-aggregator-skill 的脚本目录
# 优先查找: ~/.qoderwork/skills/news-aggregator-skill/scripts/
# 备选: ~/.agents/skills/news-aggregator-skill/scripts/

# 多源抓取
python3 {news_aggregator_dir}/scripts/fetch_news.py \
  --source {sources} \
  --keyword {keywords} \
  --deep \
  --limit 20 \
  --no-save

# 或使用日报简报
python3 {news_aggregator_dir}/scripts/daily_briefing.py --profile general
```

输出为 JSON 数组，每条包含: title, url, source, time, summary, heat_score。

### Step 2: 内容拆解

对抓取到的每条热点进行拆解分析：

```
对每条热点新闻，提取：
1. 核心事件：一句话概括发生了什么
2. 关键人物/组织：涉及的主体
3. 争议点：存在哪些对立观点或争论
4. 数据亮点：有价值的数据或对比
5. 延伸角度：可以从哪些独特角度切入（至少3个）
6. 受众画像：这个话题最吸引什么样的读者
```

### Step 3: 选题生成

从拆解结果中提炼 5-8 个选题方向。每个选题使用选题卡片模板（见 [templates/topic-card.md](../templates/topic-card.md)），包含：

- 建议标题（一句话）
- 切入角度（一段话描述怎么写）
- 写作风格建议（deep/story/sharp/light）
- 目标受众
- 适合的平台

### Step 4: 多维评分

对每个选题进行评分，维度和权重如下：

| 维度 | 说明 | 权重 | 评分标准 |
|------|------|------|---------|
| 热度 | 当前讨论/搜索热度 | 25% | 多个源同时报道=9-10; 单源热门=6-8; 小众话题=3-5 |
| 争议性 | 是否有对立观点 | 20% | 强烈对立=9-10; 中度讨论=6-8; 共识话题=3-5 |
| 时效性 | 时间窗口紧迫度 | 15% | 24h内必须写=9-10; 一周内=6-8; 常青话题=3-5 |
| 独特角度 | 差异化切入可能性 | 20% | 几乎无人写过此角度=9-10; 有少量=6-8; 烂大街=3-5 |
| 受众覆盖 | 潜在读者群大小 | 10% | 全民话题=9-10; 垂直领域=6-8; 极小众=3-5 |
| 跨平台潜力 | 多平台分发适合度 | 10% | 4平台都适合=9-10; 2-3个=6-8; 仅1个=3-5 |

综合得分 = 各维度加权平均，保留1位小数。

### Step 5: 用户选择

用 AskUserQuestion 展示 TOP 4 选题（按得分排序），每个选项格式：

```
label: "{标题} ({综合分}分)"
description: "角度: {切入角度} | 风格: {建议风格} | 平台: {适合平台}"
```

用户选择后，将选题信息传递给写作引擎。

## 策划模式（场景2）

### Step 1: 方向调研

使用 deep-research 方法论对用户给出的模糊方向进行调研：

```
调研框架：
1. 这个方向下当前最热的子话题有哪些？
2. 目标受众对哪些问题最感兴趣/最困惑？
3. 现有头部自媒体在这个方向已经覆盖了什么？留下了什么空白？
4. 这个方向有哪些可以形成系列的主线（叙事线/知识线/实操线）？
```

### Step 2: 专题方案生成

基于调研结果，生成 3-5 个专题方案：

```
每个专题方案包含：
- 专题名称（一个吸引人的系列名）
- 专题定位（一句话说明这个系列解决什么问题）
- 文章规划（3-8篇，按阅读顺序排列）
  - 每篇的标题 + 一句话摘要
- 预期受众
- 差异化优势（为什么读者要看你的而不是别人的）
```

用 AskUserQuestion 让用户选择专题方案。

### Step 3: 选题排序

用户选定专题后，按以下优先级排定写作顺序：

1. 第一篇应该是"钩子文"——最能吸引新读者的内容
2. 中间篇按逻辑递进排列
3. 最后一篇是总结/升华/行动号召

输出排序好的选题列表，每个选题附带写作风格建议。

### Step 4: 进入循环写作

将第一个选题传递给写作引擎，开始写作。每篇完成发布后，用 AskUserQuestion 询问是否继续下一篇。
