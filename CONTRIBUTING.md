# Contributing / 贡献指南

## 中文

感谢你对 Sloth-SMM-Eido 的关注！欢迎提交 Issue 和 Pull Request。

### 报告问题

请在 GitHub Issues 中提交，包含以下信息：问题描述、复现步骤、期望行为、实际行为、运行环境（操作系统、QoderWork 版本）。

### 提交改进

1. Fork 本仓库
2. 创建特性分支：`git checkout -b feature/your-feature`
3. 提交更改：`git commit -m "Add: your feature description"`
4. 推送分支：`git push origin feature/your-feature`
5. 创建 Pull Request

### 代码规范

Python 脚本遵循 PEP 8 规范，Markdown 模块文件遵循 QoderWork SKILL 标准（SKILL.md 不超过 500 行，使用渐进式披露，文件引用使用正斜杠）。

### 提交信息格式

使用 `Add:`/`Fix:`/`Update:`/`Remove:` 前缀，简要说明更改内容。

## English

Thank you for your interest in Sloth-SMM-Eido! Issues and Pull Requests are welcome.

### Reporting Issues

Please submit via GitHub Issues with: problem description, steps to reproduce, expected behavior, actual behavior, and environment details (OS, QoderWork version).

### Submitting Improvements

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Commit changes: `git commit -m "Add: your feature description"`
4. Push the branch: `git push origin feature/your-feature`
5. Create a Pull Request

### Code Style

Python scripts follow PEP 8. Markdown module files follow the QoderWork SKILL standard (SKILL.md under 500 lines, progressive disclosure, forward slashes for file references).

### Commit Message Format

Use `Add:`/`Fix:`/`Update:`/`Remove:` prefixes with a brief description.
